添加 Session

把完整 Cookie 或 sessionid 持久化到服务端。控制台生成时会自动从 Cookie 中提取 sessionid，并由服务端按 session 池自动轮换使用；OpenAI 客户端里的 API KEY 随便填就行。

POST /api/sessions

添加模型

模型管理会持久化到服务端，并同步影响 /v1/models 返回与控制台下拉。顺序 = 优先级。

POST /api/models
会作为 /v1/models 的 id 返回，也会用于图片/视频表单下拉。

图片生成

无 images 时走文生图；填写 URL 数组或上传文件时自动走图生图。控制台会请求 /api/generate/image，并由后端注入已保存的 sessionid，或从 Cookie 提取的 sessionid 进入轮换池。

POST /api/generate/image
如果同时上传文件，则优先按 multipart 图生图请求发送。

图片结果

展示返回的图片 URL 或 b64_json 摘要。URL 结果会直接渲染预览与下载入口。

等待请求
先添加 session，再提交一次图片生成请求。

视频生成

第一版主流程接 /api/generate/video。普通视频支持 5 / 10 秒；Seedance 支持 4-15 秒，并建议配合素材使用。

POST /api/generate/video
当前模型为普通视频模型，duration 建议填写 5 或 10。
Seedance 模型建议至少配合一个素材文件；国际版同步 / 异步接口先保留在“调用说明”里，不塞进第一版主表单。

视频结果

URL 结果支持直接预览与下载。b64_json 仅展示长度摘要，避免页面过重。

等待请求
先保存 session，再提交一次视频生成请求。

调用说明与示例
